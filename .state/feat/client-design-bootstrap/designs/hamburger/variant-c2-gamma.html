Implementation Notes

Complexity: MEDIUM. No spine element needed — simpler DOM than Alpha/Beta. Each bar is a flex row of segments with optional gaps. The letter is encoded by which bar(s) have gaps and where. Generic (no letter) is just 3 plain spans — the simplest possible hamburger.

CSS-only feasibility: YES. Flex segments with gap sizing. Could also be done with a single element per bar + CSS mask-image for the gaps, which would reduce DOM nodes to 3 elements total (one per bar, no segments needed).

Whitelabel API: Override --accent color. Letter encoding is a JSON config of gap positions per bar. The cleanest fallback of all three — with no letter config, it is indistinguishable from a standard hamburger icon. Best for brands that want subtlety.

Graceful degradation: This is the only variant where removing the letter config produces a standard, universally-recognized hamburger with zero visual artifacts. Alpha still has the spine; Beta still has the spine. Gamma without a letter = pure hamburger.
